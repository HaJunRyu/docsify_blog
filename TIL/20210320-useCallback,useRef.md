## 오늘 한 일과 느낀점
- 오전 11시 ~ 오후 2시 Hooks
원래는 이번주 초반부에 진작 다 읽었어야 할 Hooks파트를 예제코드를 치는 비중을 늘리다보니 오늘에서야 다 봤다. 사실 이제 다 읽었다뿐이지 사용하려고 하니 자연스럽게 써지지는 않았다. 특히 useReducer는 많이 헷갈렸고 useMemo, useCallback같은 경우는 성능 최적화와 관련된 Hook이다 보니 신경을 쓰지 않으면 잘 생각이 안나는 Hook인것 같다. 

- 오후 2시 ~ 오후 6시 쿠팡 로그인 페이지 상태관리(Lifting up state)
어제 속도가 잘 안나와서 진행하지 못했던 쿠팡 로그인 페이지의 상태를 추가해서 관리하게끔 변경했다. Lifting up state방식으로 관리하여 각각의 컴포넌트들은 직접 상태를 가지지 않고 부모 컴포넌트인 LoginPage의 상태와 그 상태를 변경하는 함수를 전달받아 사용하는 방식으로 만들어봤다. 일단 일반적인 단일 상태를 전달받는 컴포넌트들은 난이도가 쉬웠는데 하나의 상태를 가지고 같은 컴포넌트 여러개가 각자 특정 상태에 따라 상태를 전달하는 방식이 상당히 어려웠다. 사실 그냥 아무 생각없이 한다면 useState를 여러개 쓴다거나 컴포넌트 내부에서 state를 관리해줘도 되는데 지금은 Lifting up방식으로 진행하면서 hook을 사용해보는것이 목표이기 때문에 더욱 고생을 한 것 같다. input의 value같은 경우는 useReducer를 사용하여 해결하였고 input값을 검사하여 에러를 발생시키는것은 아직 마음에 드는 방법으로 해결하지 못했다. 이것 또한 reducer를 활용하면 될 것 같은데 아직 익숙치 않아서 경우가 조금만 바껴도 응용을 잘 못하는것 같다.
